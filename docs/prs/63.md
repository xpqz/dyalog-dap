This change adds a structured support intake template plus a `diagnostic-summary` CLI that summarizes diagnostic bundle JSON into actionable first-pass triage output.

## Why this change

Issue #63 addresses support workflow friction after bundle generation: maintainers still had to parse raw JSON manually and reporter intake was inconsistent.

### Issue template

Added a diagnostics-first GitHub issue template requiring:

- bundle attachment reference
- `diagnostic-summary --json` output
- platform/Dyalog version and reproduction details
- transcript artifact status

### Bundle summarizer

Added Go parser/summarizer package and CLI:

- command: `go run ./cmd/diagnostic-summary [--json] <bundle.json>`
- extracts key fields (version/workspace/diagnostic counts/transcript pointers)
- classifies probable problem area
- emits recommended next actions (missing artifacts, setup checks, redaction reminder)

### Triage docs update

Updated support triage doc with:

- first-pass summarizer usage examples
- explicit issue-template intake path

### Scaffold and tests

- new scaffold guard for template + command + parser tests
- parser tests cover classification and missing-artifact recommendation behavior
